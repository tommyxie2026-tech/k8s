---

## 1. Purpose

This document defines database conventions for CPP.

It exists to prevent naming drift, lifecycle drift and implementation-specific data model drift.

---

## 2. Table Naming

Rules:

```text
1. Table names must use plural snake_case.
2. Core resource tables use resource plural names.
3. Event tables use *_events suffix.
4. Join tables use source_target naming only when necessary.
```

Examples:

```text
clusters
storage_pools
storage_classes
vms
workflows
audit_events
resource_events
```

---

## 3. Column Naming

Rules:

```text
1. Columns use snake_case.
2. Foreign key references use <resource>_id.
3. Lifecycle shadow columns use phase.
4. JSON fields use spec/status/metadata/payload.
5. Time fields end with _at.
```

Required common columns:

```text
id
kind
name
labels
annotations
spec
status
generation
resource_version
created_at
updated_at
deleted_at
```

---

## 4. ID Convention

Rules:

```text
1. Resource IDs are UUIDv7-compatible strings.
2. IDs are generated by application/service layer, not by database autoincrement.
3. Public APIs expose IDs as strings.
4. External IDs from Kubernetes or plugins must be stored as references, not primary IDs.
```

---

## 5. Time Convention

Rules:

```text
1. Persist UTC time only.
2. API uses RFC3339-compatible timestamps.
3. UI handles local timezone conversion.
4. Do not store local timezone timestamps in database.
```

---

## 6. Lifecycle Convention

Resource lifecycle lives inside status.phase and may be mirrored to phase shadow columns for indexing.

Allowed top-level shadow column:

```text
phase
```

Avoid:

```text
state
status_text
running
is_ready
```

unless explicitly justified by a later design.

---

## 7. JSON Convention

JSON fields:

```text
labels
annotations
spec
status
metadata
payload
```

Rules:

```text
1. JSON must be valid object by default.
2. Empty object is {} not null.
3. Repositories validate required schema boundaries.
4. Unknown fields should be preserved where possible.
5. PostgreSQL JSONB migration must be considered in all designs.
```

---

## 8. Soft Delete Convention

Rules:

```text
1. deleted_at=null means active.
2. deleted_at!=null means soft deleted.
3. Normal DELETE APIs must soft delete.
4. Hard delete is only for garbage collection or explicit purge.
5. Audit events are not soft-deleted by normal resource deletion.
```

---

## 9. Resource Version Convention

Rules:

```text
1. generation increments on spec changes.
2. resource_version changes on any persisted change.
3. Update APIs must support optimistic concurrency based on resource_version.
4. Status-only updates do not increment generation but must update resource_version.
```

---

## 10. Repository Convention

Rules:

```text
1. No router may issue SQL.
2. No plugin may issue SQL.
3. No executor may issue SQL.
4. Service layer calls repository interfaces.
5. Repositories own dialect-specific behavior.
6. Repository methods must be testable without running a full API server.
```

Recommended methods:

```text
create(resource)
get(id)
list(filter)
update_spec(id, spec, resource_version)
update_status(id, status)
soft_delete(id, resource_version)
```

---

## 11. Event Convention

Rules:

```text
1. Resource lifecycle changes emit ResourceEvent.
2. Security and user operations emit AuditEvent.
3. Events are append-only.
4. Events include actor where available.
5. Events include target_kind and target_id where applicable.
```

---

## 12. Constraint Convention

Rules:

```text
1. Primary key is id.
2. Foreign keys are recommended where SQLite/PostgreSQL compatibility allows.
3. Critical uniqueness must be enforced at repository level in V2.0.
4. PostgreSQL production mode may add stronger partial unique indexes.
```

---

## 13. Index Convention

Rules:

```text
1. Index cluster_id on cluster-scoped resources.
2. Index phase for frequently queried lifecycle state.
3. Index created_at for pagination.
4. Index target_kind + target_id for workflows, backups and events.
5. Avoid premature JSON indexes in SQLite.
```

---

## 14. Review Items

Before freezing P2-05, confirm:

```text
1. plural snake_case table naming.
2. UUIDv7-compatible app-generated IDs.
3. UTC/RFC3339 time convention.
4. status.phase as lifecycle convention.
5. repository-only database access.
6. append-only event model.
7. soft delete by default.
```
